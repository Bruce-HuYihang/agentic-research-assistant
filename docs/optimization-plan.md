## 1. 研究生痛点分析

研究生写论文的典型痛点，以及当前助手能解决多少：

| 痛点 | 现版本 | 优化后 |
|------|--------|--------|
| 不知道研究方向有哪些新进展 | ❌ 只能搜百度百科 | ✅ 自动扫 arXiv + 顶会论文 |
| 文献太多读不完 | ❌ 不支持 | ✅ 批量导入，自动摘要 |
| 写 Related Work 头疼 | ❌ 不支持 | ✅ 给定方向，自动生成 |
| 引用格式搞错被导师骂 | ❌ 不支持 | ✅ 自动 BibTeX |
| 想找某篇论文的具体方法 | ❌ 不支持 | ✅ 语义检索论文库 |
| 开题报告没思路 | ❌ 不支持 | ✅ 基于文献自动生成大纲 |
| 中英文混搜麻烦 | ❌ 仅中文搜索 | ✅ 中英双语自动搜索 |
| 实验不知道怎么做 | ❌ 不支持 | ✅ 根据论文自动提取实验方法 |
| 论文写完查漏补缺 | ❌ 不支持 | ✅ 检查引用完整性 + 盲区分析 |

## 3. P0 — 必须实现

### 3.1 学术搜索源接入

**目标**：能搜到论文，而不是百度百科。

**具体功能**：
- 接入 **arXiv API**（免费，无需 Key）— 搜计算机、物理、数学预印本
- 接入 **Semantic Scholar API**（免费，需注册 Key）— 搜全领域论文，含引用关系
- 接入 **Google Scholar**（通过爬虫或第三方）— 搜所有学术资料
- 保留百度搜索作为中文补充

**使用方式**：
```
用户：Transformer 在计算机视觉中的应用
→ Agent 自动调用 arXiv + Semantic Scholar 搜索英文论文
→ 同时百度搜索中文资料
→ 混合生成报告
```

**用户感知**：报告中的「来源」从百度链接变成论文标题 + arXiv/DOI 链接。

**预估工作量**：1-2 天

---

### 3.2 上传 PDF 分析

**目标**：能拖入论文 PDF，Agent 读懂并帮你总结。

**具体功能**：
- 在 Streamlit 界面添加文件上传组件
- 支持单篇 PDF 上传（支持 arXiv PDF 链接）
- Agent 读取全文后：
  - 生成摘要（中文）
  - 提取核心方法/模型
  - 提取实验数据集 + 指标
  - 提取主要结论
- 支持上传后追问：「这篇论文用了什么数据集？跟那篇对比有什么优势？」

**用户感知**：

```
[上传文件] my_paper.pdf
→ Agent 读完了，这是摘要：
该论文提出了一种...
核心方法: ...
数据集: CIFAR-10, ImageNet
指标: Accuracy 95.2%
```

**需要注意**：PyMuPDF 已支持，但长论文（>20 页）需要处理 token 限制，可以用摘要-分块-再整合的策略。

**预估工作量**：2-3 天

---

### 3.3 引用格式导出

**目标**：写论文时可以直接复制引用。

**具体功能**：
- 报告末尾自动生成引用列表
- 支持三种格式切换：**BibTeX** / **GB/T 7714（国标）** / **APA**
- 每条引用包含：作者、标题、期刊/会议、年份、DOI/URL
- 支持一键复制到剪贴板

**用户感知**：

```
引用格式: [BibTeX ▼]

@article{smith2025quantum,
  title={Quantum Computing in Finance},
  author={Smith, J. and Wang, L.},
  journal={Nature},
  year={2025}
}
[📋 复制]
```

**预估工作量**：0.5 天

## 4. P1 — 强烈建议

### 4.1 文献批量导入与综述生成

**目标**：一个方向丢一堆论文进去，Agent 帮你读完后生成文献综述。

**具体功能**：
- 批量上传 PDF（拖入多篇）
- 或者输入一个主题，Agent 自动搜索相关论文并下载
- Agent 逐篇阅读，抽取：
  - 每篇论文的核心信息（方法、数据集、指标）
  - 论文之间的引用关系
  - 共识与分歧
- 生成结构化文献综述表格：

| 论文 | 方法 | 数据集 | 指标 | 发表年份 |
|------|------|--------|------|----------|
| Smith et al. | CNN | CIFAR-10 | 92% | 2024 |
| Wang et al. | Transformer | ImageNet | 95% | 2025 |

- 输出综述报告，包含每条发现对应的文献引用

**使用场景**：开题前快速了解某个领域的研究现状。

**预估工作量**：3-4 天

---

### 4.2 Related Work 章节生成

**目标**：给定研究方向，直接输出论文的 Related Work 草稿。

**具体功能**：
- 输入：你论文的题目 + 摘要 + 关键词
- Agent 搜索相关文献
- 自动分类组织：
  - 早期工作
  - 近期主流方法（分流派）
  - 各方法的优缺点
  - 你的工作与已有工作的关系（区别/改进）
- 输出可直接放进论文的 Related Work 章节
- 可调整输出风格（期刊/会议/简洁/详细）

**使用场景**：写论文最头疼的部分，尤其是对新手。

**预估工作量**：2-3 天

---

### 4.3 多轮对话追问

**目标**：研究完成后可以继续追问，而不是只能查一次。

**具体功能**：
- 支持基于上一次搜索结果的追问
- 例如：
  - 用户第一轮：「量子计算在金融中的应用」
  - 用户追问：「第三篇文章的方法和第一篇有什么区别？」
  - Agent 基于已有资料回答，必要时补充搜索
- 需要维护对话历史上下文（当前 session.py 只有 JSON 保存，需要增强）

**使用场景**：读了一堆资料后想深入某个点。

**预估工作量**：2 天

## 5. P2 — 锦上添花

### 5.1 中英文双语搜索

**目标**：一个中文问题，自动从中英文角度各搜一轮。

**具体功能**：
- 用户输入中文问题
- 第一轮：英文搜索（arXiv / Semantic Scholar）
- 第二轮：中文搜索（百度）
- 报告中区分「英文文献」和「中文资料」
- 自动翻译英文摘要为中文

**预估工作量**：1 天

---

### 5.2 论文推荐引擎

**目标**：根据当前研究内容，推荐相关未读论文。

**具体功能**：
- 基于当前搜索结果的语义，用 ChromaDB 从已读论文库中找相似论文
- 但更有用的是：调用 Semantic Scholar 的 Recommendations API
- 推荐理由：为什么推荐这篇（方法相似 / 引用相关 / 同一作者）

**预估工作量**：1 天

---

### 5.3 Notion / Zotero 导出

**目标**：和你的文献管理工具打通。

**具体功能**：
- 一键导出到 Notion 数据库（通过 Notion API）
- 一键导出 RIS 格式（Zotero 支持的通用格式）
- 推送到 Obsidian / Markdown 文件夹

**预估工作量**：1-2 天（取决于要接几个平台）

---

### 5.4 开题报告辅助

**目标**：输入研究领域，自动生成开题报告大纲。

**具体功能**：
- 输入：研究领域 / 关键词
- 输出：
  - 选题背景（基于最新文献）
  - 国内外研究现状（结构化综述）
  - 存在的问题和不足
  - 拟研究内容
  - 技术路线
  - 预期成果
- 每个部分都基于真实文献，标注引用

**预估工作量**：1 天（依赖文献批量导入做前置）

---

### 5.5 实验方法提取

**目标**：读论文时自动提取实验设置，方便复现。

**具体功能**：
- 上传论文 → Agent 提取实验部分：
  - 数据集
  - 训练配置（batch size / learning rate / epoch）
  - 硬件环境（GPU / 内存）
  - 对比方法
  - 评估指标
- 输出可直接参考的实验设置清单

**预估工作量**：1 天

---

### 5.6 引用完整性检查

**目标**：写完后检查论文引用是否全面。

**具体功能**：
- 输入你的论文草稿
- Agent 检查：
  - 关键论断是否有文献支撑
  - 是否遗漏了该领域的经典文献
  - 引用格式是否一致
- 输出缺失引用建议

**预估工作量**：1 天

## 6. 完整研途场景覆盖

一条时间线看这个助手怎么贯穿研究生生涯：

### 研一上：探索方向

```
你：我在 AI + 金融方向，帮我看看这个领域有什么研究热点？
助手：→ 搜 arXiv + Semantic Scholar
       → 生成领域概览报告
       → 推荐 10 篇必读论文
       → 列出几个热门子方向
```

### 研一下：深入文献

```
你：《Attention Is All You Need》
助手：→ 上传 PDF（或输入 arXiv ID）
       → 生成精读笔记
       → 追问：「这篇论文的实验设置是什么样的？」
       → 追问：「它有什么后续改进工作？」
```

### 研二上：开题 / 写小论文

```
你：帮我写 Related Work，方向是「时间序列预测中的 Transformer」
助手：→ 搜索相关文献
       → 自动分类：早期RNN / LSTM方法 → Transformer方法 → 最新改进
       → 生成 Related Work 章节初稿
       → 导出 BibTeX 引用列表
```

### 研二下/研三：写大论文

```
你：检查一下我这篇论文的引用完整性
助手：→ 读取你的草稿
       → 发现你没引 Domain Adaptation 的几篇经典论文
       → 补充推荐
       → 生成文献综述表
```
